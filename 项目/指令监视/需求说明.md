# 背景

你是一名高级java开发工程师，现在需要完善补充一项后端开发的基本逻辑的补充工作，背景为：对卫星进行发送指令需要进行实时监视，通过下发的一些状态参数进行判读，指令是否执行成功，具体需求如下：

# 需求：

1、使用java编写代码 + mysql作为数据库 ，redis作为nosql数据库

2、利用SpringBoot+SpringJPA作为后端技术栈

3、指令包括几种状态：未执行，已编制，执行中，已完成，执行失败等，需要实时获取当前状态，其中未执行和已编制需要根据时间去查库，执行中会在kafka中实时获取，已完成代表，下行收到的遥测参数满足要求，执行失败代表在时间范围内有部分遥测参数没有满足要求，或者没有在kafka中获取到执行的此条指令

4、一条指令可能需要0个或者多个遥测参数进行判断，遥测参数要有判断公式，判断依据已经提前配置好，只有关联的所有判定都满足要求才能将指令转变为已完成

5、指令有执行时间，同时约定执行指令后的时间范围，只要时间范围内，遥测参数都满足要求才可以判定已完成

6、如果在时间范围内未完成则判定为未完成

7、指令与遥测参数的关联性和遥测参数的判断依据已经写在了excel表格里，会直接存储到数据库

8、已经有Kafka消费者，只需要集成状态判断逻辑

9、使用您提到的状态：未执行、已编制、执行中、已完成、执行失败

10、 每个指令可配置不同的时间范围

11、需要支持复杂表达式

12、需要注意基本流程需要通过websocket进行实时实时推送，bing
