## 🧪 实验变量说明：

| 参数名                 | 含义（推测）                             |
| ---------------------- | ---------------------------------------- |
| `learning_rate`        | 学习率，控制模型权重更新速度             |
| `cluster_temperature`  | 聚类温度，控制聚类 logits 的平滑程度     |
| `instance_temperature` | 实例温度，控制样本之间对比损失的平滑程度 |
| `threshold`            | 阈值，可能控制正负样本划分或伪标签可信度 |
| `loss`                 | 损失函数值，越低表示模型越优             |

## 🔍 分析角度一：固定其它参数，观察单个参数的影响

### 1️⃣ **固定 learning_rate = 0.0005，cluster_temperature = 0.8，threshold = 0.95**

#### 看 `instance_temperature` 的变化与 loss 的关系：

| instance_temperature | loss |
| -------------------- | ---- |
| 0.01                 | 0.44 |

| 0.3                  | 1.54 |
| 0.4                  | 1.71 |

✅ **结论**：
 `instance_temperature` 越低，`loss` 趋势越小，说明模型更容易收敛。

➡️ **优化建议**：
 尝试更低的 `instance_temperature`，如 `0.005`、`0.02` 等，进行细化搜索。

------

### 2️⃣ **固定 instance_temperature = 0.03，cluster_temperature = 0.8，threshold = 0.95**

#### 看 `learning_rate` 对 loss 的影响：

| learning_rate | loss      |
| ------------- | --------- |
| 0.0005        | 0.48      |
| 0.001         | 0.52      |
| 0.01          | 0.74~0.76 |

✅ **结论**：
 较低的 `learning_rate`（如 0.0005）带来更小的 `loss`。太大会导致不稳定。

➡️ **优化建议**：
 保持小学习率或使用 **warm-up + cosine decay** 的学习率调度器。

### 3️⃣ **固定 learning_rate 和 instance_temperature**

➡️ **优化建议**：
